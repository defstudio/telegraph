Этот tutorial проводит через полный happy path: установить Telegraph в Laravel-приложение, создать bot в BotFather, зарегистрировать его в Telegraph, открыть webhook, обработать команду `/start`, зарегистрировать chat и проверить поведение тестом без реальных Telegram-запросов.

## Prerequisites

Нужны:

- Laravel-приложение на PHP 8.2+.
- Публичный HTTPS URL для webhook. Для local development используйте tunnel вроде `ngrok`, `expose` или аналогичный HTTPS endpoint.
- Telegram account с доступом к [@BotFather](https://t.me/botfather).

Не коммитьте реальные bot tokens, webhook secret tokens или dumps входящих Telegram payloads. Храните секреты в `.env` или в secret manager вашего окружения.

## 1. Install Telegraph

Установите package в host Laravel app:

```shell
composer require defstudio/telegraph
```

Опубликуйте config и migrations:

```shell
php artisan vendor:publish --tag="telegraph-config"
php artisan vendor:publish --tag="telegraph-migrations"
php artisan migrate
```

Проверьте базовые значения в `.env`:

```dotenv
APP_URL=https://example.test
TELEGRAM_WEBHOOK_DOMAIN=https://example.test
TELEGRAPH_BOT_TOKEN=replace-with-botfather-token
TELEGRAPH_WEBHOOK_SECRET=replace-with-a-random-secret
```

`TELEGRAPH_BOT_TOKEN` нужен только если ваше приложение читает token из config вместо interactive `telegraph:new-bot` wizard. `TELEGRAM_WEBHOOK_DOMAIN` нужен, если URL webhook должен отличаться от `APP_URL`. `TELEGRAPH_WEBHOOK_SECRET` используйте как значение для `--secret`, чтобы Telegram отправлял его в header `X-Telegram-Bot-Api-Secret-Token`; используйте случайную строку и не публикуйте её в logs.

Если вы создаёте bot программно, добавьте token в application config, например в `config/services.php`:

```php
'telegram' => [
    'bot_token' => env('TELEGRAPH_BOT_TOKEN'),
],
```

## 2. Create a Telegram bot

Откройте [@BotFather](https://t.me/botfather), отправьте `/newbot`, задайте name и username, затем сохраните выданный token в защищённом месте.

Зарегистрируйте bot в Telegraph:

```shell
php artisan telegraph:new-bot
```

Команда попросит bot token и readable name. Если это первый bot в приложении, wizard также может предложить создать chat и сразу настроить webhook. Если вы уже знаете numeric Telegram `chat_id`, можно создать chat в wizard сразу. Если нет, пропустите chat на этом шаге и получите `chat_id` позже через временный onboarding режим для unknown chats.

Если вы создаёте bot программно, используйте `TelegraphBot`:

```php
use DefStudio\Telegraph\Models\TelegraphBot;

$bot = TelegraphBot::create([
    'token' => config('services.telegram.bot_token'),
    'name' => 'Support bot',
]);
```

Подробнее о создании bot в BotFather и privacy mode: [Creating a new bot](1.new-bot).

## 3. Register the webhook

Telegram доставляет updates только на публичный HTTPS URL. Для local development сначала поднимите tunnel и обновите URL в `.env`:

```dotenv
APP_URL=https://your-tunnel.example
TELEGRAM_WEBHOOK_DOMAIN=https://your-tunnel.example
```

После изменения `.env` в приложениях с cached config выполните:

```shell
php artisan config:clear
```

Зарегистрируйте webhook:

```shell
php artisan telegraph:set-webhook
```

Если в приложении несколько bots, передайте ID нужного bot:

```shell
php artisan telegraph:set-webhook 1
```

Для безопасного webhook и чистого первого запуска можно передать options:

```shell
export TELEGRAPH_WEBHOOK_SECRET="replace-with-a-random-secret"
php artisan telegraph:set-webhook 1 --drop-pending-updates --secret="${TELEGRAPH_WEBHOOK_SECRET}"
```

Laravel `.env` не экспортирует переменные в shell автоматически: если используете shell substitution `${TELEGRAPH_WEBHOOK_SECRET}`, выполните `export` в текущей shell-сессии или передайте явное placeholder-значение вместо substitution. В shared environments учитывайте, что CLI option может быть виден в process list во время выполнения команды.

Проверить состояние webhook можно так:

```shell
php artisan telegraph:debug-webhook 1
```

Не публикуйте command output, если в нём есть URL, secret-dependent diagnostics или данные пользователей. Детали регистрации webhook: [Register Webhooks](../15.webhooks/2.registering-webhooks).

## 4. Handle `/start`

Создайте custom handler в host app:

```php
<?php

namespace App\Http\Webhooks;

use DefStudio\Telegraph\Handlers\WebhookHandler;

class FirstBotWebhookHandler extends WebhookHandler
{
    public function start(): void
    {
        $this->chat
            ->html('Hello! Telegraph is connected.')
            ->send();
    }
}
```

Зарегистрируйте handler в `config/telegraph.php`:

```php
'webhook' => [
    // ...
    'handler' => App\Http\Webhooks\FirstBotWebhookHandler::class,
],
```

Когда Telegram отправляет message text `/start`, `WebhookHandler` распознаёт command и вызывает public method `start()`. Method должен быть `public`; protected/private methods не вызываются как commands.

> [!NOTE]
> Для local diagnostics можно временно включить `TELEGRAPH_WEBHOOK_DEBUG=true`, но не логируйте raw tokens, webhook secrets или полный Telegram payload в shared environments.

## 5. Register a chat

По умолчанию Telegraph отклоняет messages и commands из неизвестных chats через `telegraph.security.allow_messages_from_unknown_chats=false`. Поэтому `/chatid` сработает только для уже зарегистрированного chat или после временного onboarding-разрешения unknown chats.

Если вы уже знаете numeric Telegram `chat_id`, пропустите временное разрешение и сразу выполните `telegraph:new-chat`. Если `chat_id` неизвестен, временно включите unknown messages в `config/telegraph.php`:

```php
'security' => [
    // ...
    'allow_messages_from_unknown_chats' => true,
],
```

После изменения config в приложениях с cached config выполните:

```shell
php artisan config:clear
```

Теперь отправьте вашему bot команду:

```text
/chatid
```

Базовый `WebhookHandler` уже содержит public method `chatid()`, поэтому custom handler, который extends `WebhookHandler`, тоже ответит на `/chatid`.

Зарегистрируйте chat:

```shell
php artisan telegraph:new-chat 1
```

Если bot один, аргумент можно не указывать:

```shell
php artisan telegraph:new-chat
```

После успешной регистрации chat верните безопасное значение:

```php
'security' => [
    // ...
    'allow_messages_from_unknown_chats' => false,
],
```

И снова очистите config cache:

```shell
php artisan config:clear
```

Теперь приложение может отправлять исходящие сообщения в этот chat:

```php
use DefStudio\Telegraph\Models\TelegraphChat;

$chat = TelegraphChat::query()
    ->where('name', 'Support chat')
    ->firstOrFail();

$chat->html("<strong>Hello!</strong>\n\nI'm here!")->send();
```

Webhook reply на `/start` отправляется как реакция на входящий update. `$chat->html(...)->send()` - это исходящее сообщение из вашего приложения, например из controller, command или job.

Если onboarding должен принимать unknown chats дольше одного setup-шага, настройте security flags осознанно и прочитайте [Handle requests from unknown chats](../15.webhooks/1.overview#handle-requests-from-unknown-chats).

## 6. Test the handler

В feature test можно проверить handler без сетевых запросов через `Telegraph::fake()`:

```php
<?php

use App\Http\Webhooks\FirstBotWebhookHandler;
use DefStudio\Telegraph\Facades\Telegraph;
use DefStudio\Telegraph\Models\TelegraphBot;
use DefStudio\Telegraph\Telegraph as TelegraphCore;
use Illuminate\Http\Request;

it('handles /start command', function () {
    $bot = TelegraphBot::create([
        'token' => 'test-bot-token',
        'name' => 'Support Bot',
    ]);

    $chat = $bot->chats()->create([
        'chat_id' => '-123456789',
        'name' => 'Personal chat',
    ]);

    Telegraph::fake();

    app(FirstBotWebhookHandler::class)->handle(
        Request::create('', 'POST', [
            'message' => [
                'message_id' => 123456,
                'chat' => [
                    'id' => (int) $chat->chat_id,
                    'type' => 'private',
                    'username' => 'john-smith',
                ],
                'text' => '/start',
                'date' => 1646516736,
            ],
        ]),
        $bot,
    );

    Telegraph::assertSent('Hello! Telegraph is connected.');
    Telegraph::assertSentData(TelegraphCore::ENDPOINT_MESSAGE, [
        'chat_id' => $chat->chat_id,
        'text' => 'Hello! Telegraph is connected.',
    ]);
});
```

Такой test создаёт persisted `TelegraphBot` и первый known `TelegraphChat`, включает fake-режим до обработки webhook и проверяет, что handler отправляет сообщение именно в chat из incoming update. `Telegraph::fake()` не делает реальных Telegram requests; при ручной отладке sent payloads можно посмотреть через `Telegraph::dumpSentData()`.

Подробнее о fake/assertions: [Fake Telegram interaction](../17.testing/1.fake) и [Assertions](../17.testing/2.assertions).

## Troubleshooting

| Symptom | Fix |
| --- | --- |
| `telegraph:new-bot` не принимает token | Скопируйте token из BotFather заново и не добавляйте spaces или quotes. Не вставляйте token в docs, tickets или logs. |
| Webhook не вызывается локально | Убедитесь, что tunnel URL публичный, HTTPS и совпадает с `APP_URL` или `TELEGRAM_WEBHOOK_DOMAIN`. После смены tunnel URL выполните `php artisan config:clear` и снова `telegraph:set-webhook`. |
| Telegram продолжает слать старые updates | Запустите `php artisan telegraph:set-webhook 1 --drop-pending-updates`. |
| Несколько bots в приложении | Передавайте `{bot_id}` в `telegraph:set-webhook`, `telegraph:debug-webhook` и `telegraph:new-chat`. |
| `/start` не вызывает method | Проверьте, что handler зарегистрирован в `config('telegraph.webhook.handler')`, method называется `start`, он `public`, а config cache очищен. |
| Chat не найден | Если `chat_id` неизвестен, временно включите `allow_messages_from_unknown_chats`, получите ID через `/chatid`, зарегистрируйте chat через `telegraph:new-chat` и верните безопасную config. |
| `telegraph:debug-webhook` показывает старый URL | Очистите config cache, обновите `APP_URL`/`TELEGRAM_WEBHOOK_DOMAIN` и повторно зарегистрируйте webhook. |

Следующий шаг после happy path - добавить реальные commands, keyboards или callback handling. Начните с [Webhook request types](../15.webhooks/4.webhook-request-types) и [Message Keyboards](../12.features/3.keyboards).
