Чтобы получать Telegram updates через webhook, нужно зарегистрировать публичный `https` URL для конкретного bot. Для быстрого onboarding path смотрите [quickstart webhook step](../11.quickstart/3.setting-webhook); эта страница описывает параметры регистрации и диагностику подробнее.

## Public URL и local tunnel

Telegram должен достучаться до вашего приложения извне. В production это обычно домен приложения, а в local development - временный `https` tunnel:

```shell
ngrok http 8000
```

Скопируйте публичный tunnel URL и настройте один из вариантов:

```dotenv
APP_URL=https://example.ngrok-free.app
TELEGRAM_WEBHOOK_DOMAIN=https://example.ngrok-free.app
```

`TELEGRAM_WEBHOOK_DOMAIN` задаёт `telegraph.webhook.domain` и используется при регистрации webhook вместо домена из `APP_URL`. Это удобно, когда остальное приложение остаётся на локальном `APP_URL`, но Telegram должен получать публичный route.

Если вы используете `php artisan config:cache`, после изменения `.env` выполните:

```shell
php artisan config:clear
```

## Route URL

Telegraph регистрирует Laravel route с именем `telegraph.webhook`. По умолчанию путь такой:

```text
/telegraph/{token}/webhook
```

Полный URL при стандартном config будет выглядеть так:

```text
https://example.ngrok-free.app/telegraph/{token}/webhook
```

Путь можно изменить через `TELEGRAPH_WEBHOOK_URL` (`telegraph.webhook.url`) или отключить route, задав значение `null` в config:

```dotenv
TELEGRAPH_WEBHOOK_URL=/telegram/{token}/webhook
```

Если route меняется, проверьте, что middleware и reverse proxy пропускают `POST` requests на новый URL.

## Secret token

Telegram Bot API позволяет передать `secret_token` при регистрации webhook. Telegram будет отправлять это значение в header `X-Telegram-Bot-Api-Secret-Token` для последующих webhook requests.

Храните значение в `.env` и не публикуйте его в issue, logs или screenshots:

```dotenv
TELEGRAPH_WEBHOOK_SECRET=your-secret-token
```

Для CLI registration передайте secret явно:

```shell
php artisan telegraph:set-webhook {bot_id} --secret="${TELEGRAPH_WEBHOOK_SECRET}"
```

Для programmatic registration передайте `secretToken`:

```php
/** @var DefStudio\Telegraph\Models\TelegraphBot $telegraphBot */

$telegraphBot->registerWebhook(
    secretToken: config('telegraph.webhook.secret'),
)->send();
```

> [!NOTE]
> Telegraph передаёт `secret_token` в Telegram при регистрации. Если приложению нужна обязательная inbound-проверка header `X-Telegram-Bot-Api-Secret-Token`, добавьте её в middleware из `telegraph.webhook.middleware` или в собственный request handling layer.

## Artisan command

Зарегистрировать webhook можно командой:

```shell
php artisan telegraph:set-webhook {bot_id}
```

`{bot_id}` можно опустить, если в системе только один bot. Доступные options:

```shell
php artisan telegraph:set-webhook {bot_id} \
    --drop-pending-updates \
    --max-connections=100 \
    --secret=your-secret-token
```

- `--drop-pending-updates` удаляет pending updates на стороне Telegram при смене webhook.
- `--max-connections` задаёт maximum allowed simultaneous connections. Значение по умолчанию берётся из `telegraph.webhook.max_connections`.
- `--secret` передаёт Telegram `secret_token`.

## Allowed updates

`telegraph.webhook.allowed_updates` можно задать в config, если бот должен получать только часть update types. Значение `null` оставляет поведение Telegram по умолчанию: все update types, кроме некоторых opt-in events вроде `chat_member`, `message_reaction` и `message_reaction_count`.

Пример config:

```php
'webhook' => [
    'allowed_updates' => ['message', 'callback_query'],
],
```

## Programmatic registration

Если у приложения есть собственная bot management logic, webhook можно зарегистрировать через `TelegraphBot`:

```php
/** @var DefStudio\Telegraph\Models\TelegraphBot $telegraphBot */

$telegraphBot->registerWebhook(
    dropPendingUpdates: true,
    maxConnections: 100,
    secretToken: 'your-secret-token',
)->send();
```

Подробнее о сигнатуре метода смотрите в [TelegraphBot documentation](../14.models/1.telegraph-bot#registerWebhook).

> [!WARNING]
> Manual updates polling недоступен, пока для bot настроен webhook. Сначала удалите webhook через метод [unregisterWebhook](3.deleting-webhooks#programmatically).

## Проверка через debug command

После регистрации проверьте состояние webhook на стороне Telegram:

```shell
php artisan telegraph:debug-webhook {bot_id}
```

Команда показывает Telegram webhook diagnostics, например `url`, `pending_update_count`, `last_error_date`, `last_error_message`, `max_connections` и `allowed_updates`, если Telegram вернул эти поля. Это диагностика Telegram delivery, а не dump входящего Laravel request.

Для runtime diagnostics используйте:

- `php artisan telegraph:debug-webhook {bot_id}` для состояния webhook в Telegram.
- `storage/logs/laravel.log` для ошибок handler, middleware и application code.
- tunnel inspector (`ngrok` web UI или аналог) для проверки входящих `POST` requests.

Не вставляйте реальные bot tokens, webhook secrets, Telegram headers или полные user payloads в публичные issues и shared logs.

## Troubleshooting local development

### Telegram отклоняет URL

Проверьте, что webhook URL начинается с `https://`. Если используется route без `TELEGRAM_WEBHOOK_DOMAIN`, Laravel должен генерировать secure URL из `APP_URL`.

### Tunnel URL изменился после перезапуска

Бесплатные tunnel URLs часто меняются при restart. Обновите `APP_URL` или `TELEGRAM_WEBHOOK_DOMAIN`, выполните `php artisan config:clear`, затем снова запустите `telegraph:set-webhook`.

### `debug-webhook` показывает старый URL

Скорее всего config cache держит старые env values или webhook не был перерегистрирован после изменения tunnel. Выполните:

```shell
php artisan config:clear
php artisan telegraph:set-webhook {bot_id} --secret=your-secret-token
php artisan telegraph:debug-webhook {bot_id}
```

### Route отвечает `404`

Проверьте `TELEGRAPH_WEBHOOK_URL` и опубликованный config. По умолчанию route должен быть `/telegraph/{token}/webhook`; если вы переопределили path, тот же path должен быть зарегистрирован в Telegram. Также проверьте `php artisan route:list` в host application.

### Route отвечает `401` или `403`

Проверьте middleware из `telegraph.webhook.middleware`, CSRF/external access rules, reverse proxy и любую custom-проверку `X-Telegram-Bot-Api-Secret-Token`. Secret в Telegram registration и expected value в middleware должны совпадать.

### `/chatid` не отвечает в новом чате

По умолчанию webhook handler принимает requests только от известных `TelegraphChat` models и может отклонять unknown chats. Для получения первого `chat_id` используйте default handler flow из [overview](1.overview#default-handler) или настройте security options для unknown chats осознанно.

### Pending updates приходят после смены webhook

Если старые updates больше не нужны, перерегистрируйте webhook с `--drop-pending-updates`:

```shell
php artisan telegraph:set-webhook {bot_id} --drop-pending-updates --secret=your-secret-token
```

Если webhook нужно удалить полностью, используйте `telegraph:unset-webhook`.
